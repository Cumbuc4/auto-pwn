# Contribuindo

Obrigado por considerar contribuir com este projeto.

## Como contribuir

1. Faca um fork do repositorio.
2. Crie uma branch para sua mudanca:
   `git checkout -b minha-mudanca`
3. Fa√ßa as alteracoes e mantenha o codigo claro e consistente.
4. Atualize o `README.md` se houver mudancas de uso.
5. Abra um Pull Request descrevendo:
   - o que mudou
   - o motivo da mudanca
   - como testar (se aplicavel)

## Boas praticas

- Prefira mudancas pequenas e focadas.
- Evite inserir dados sensiveis ou privados.
- Mantenha compatibilidade com Linux e Python 3.
- Use linguagem respeitosa nas discussoes.

## Relato de problemas

Ao abrir uma issue, inclua:

- passos para reproduzir
- comportamento esperado vs atual
- logs relevantes
- versao do sistema e ferramentas
